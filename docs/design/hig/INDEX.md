# Apple HIG Catalog — Liquid Glass Design Audit Reference

Local, cited catalog of the Apple Human Interface Guidelines pages relevant to auditing a Liquid-Glass web UI's visual chrome. Each page below was fetched from Apple's DocC JSON content endpoint (`developer.apple.com/tutorials/data/design/human-interface-guidelines/<slug>.json`), which returns the real page content the JS single-page app renders, and distilled to markdown in this directory.

**developer.apple.com was fully reachable** through the outbound proxy via the JSON endpoint. All 52 target pages returned HTTP 200 and were cataloged with zero fetch failures. (The human-facing HTML URLs are JS-rendered shells that return no readable body to a plain fetch/WebFetch; the JSON endpoint is the reliable source and is what every file cites at the top, alongside the human-readable URL.)

### Scope

This catalog is deliberately scoped to the **visual-chrome / design-relevant** HIG set for a Liquid-Glass audit. See "Deliberately skipped" at the bottom for the platform/commerce pages intentionally excluded.

The **Status** column marks whether a page is **NET-NEW** to the Applicant corpus or was **already-in-corpus** (already present under `docs/design/liquid-glass/sources/`, per coordinator guidance). Already-in-corpus pages are cataloged here too for a single complete reference table, but the net-new pages are the additive value of this crawl.

## Deliberately skipped

A literal crawl of *every* HIG subpage includes many pages irrelevant to auditing a Liquid-Glass web UI's visual chrome. To stay within budget and signal, the following categories were intentionally **not** cataloged:

- **Platform-specific experience pages** with no web-chrome analog: tvOS (focus & selection, top shelf, remote), watchOS (complications, watch faces, Digital Crown, always-on), CarPlay, visionOS-only spatial pages (eyes, immersive experiences, spatial layout), and Apple TV/Home.
- **Commerce & account/business flows**: App Store product pages, in-app purchase, subscriptions, offer codes, Apple Pay & Wallet, Sign in with Apple (the auth flow itself), Tap to Pay, Game Center, ratings and reviews.
- **Hardware & input modalities** not applicable to a web UI: game controllers, gyroscope/accelerometer, Digital Crown, Apple Pencil, remotes, keyboards (physical shortcuts), Nearby interactions, NFC, Live Photos capture, camera controls.
- **Games & media production**: game-design guidance, augmented reality, HDR/EDR imaging pipelines, spatial audio, video playback engineering specifics beyond the on-screen controls already covered.
- **Technologies / system-integration pages** that are engine/OS features rather than visual chrome: Siri, App Shortcuts/App Intents, Live Activities engineering, CallKit, HealthKit, HomeKit, iMessage apps & stickers, Handoff, Widgets *engineering* (only the visual bits of Widgets were kept), Maps, ML/AI features, ShazamKit, machine-learning.
- **Writing / non-visual foundations** beyond the audit's chrome focus: 'Writing' (the general voice-and-tone page) and 'Terminology' were not fetched; the visually-relevant foundations (Color, Materials, Typography, Layout, Motion, Dark Mode, Accessibility, Right to left, Inclusion, Branding) were.

If any skipped page later proves relevant to the audit, it can be added with the same JSON-endpoint method used here.
